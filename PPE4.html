Serveur PXE

Le boot PXE pour Preboot Execution Environment est un protocole de démarrage réseau qui permet à un ordinateur local de démarrer à partir de données situées sur un serveur distant, plutôt que de démarrer à partir du contenu de la clé USB.

Si l'on a besoin d'installer Windows 11 sur 20 machines différentes, plutôt que d'utiliser 20 clés USB ou de faire les machines une par une si l'on a qu'une seule clé USB, on va effectuer un boot PXE sur chaque machine et elles vont récupérer les sources d'installation depuis notre serveur.

Contexte et Compétences mises en oeuvre

Contexte de ce Projet:

Le déploiement de systèmes d'exploitation sur plusieurs machines peut être fastidieux et chronophage, en particulier lorsqu'il s'agit de configurer chaque machine individuellement à l'aide de clés USB ou de supports physiques. Dans ce contexte, le boot PXE (Preboot Execution Environment) offre une solution efficace en permettant le démarrage des machines à partir d'un serveur distant via le réseau.

L’organisation de projet:

  • Moyens et Techniques :
    1. Utilisation du rôle de service de déploiement Windows (WDS) de Microsoft Windows Server pour le serveur PXE.
    2. Création d'images d'installation personnalisées pour les différents systèmes d'exploitation à déployer. Ces images sont préparées avec les logiciels, pilotes et configurations spécifiques nécessaires pour chaque environnement utilisateur.
    3. Configuration du BIOS des machines clientes pour activer le démarrage PXE.

    En résumé, la mise en place d'un serveur PXE nécessite des compétences approfondies en administration système, réseaux informatiques et déploiement de systèmes d'exploitation. Cette solution offre une gestion centralisée et efficace des installations sur un grand nombre de machines, ce qui représente un avantage significatif en termes de gain de temps et de productivité dans les environnements informatiques d'entreprise.

    1. Objectifs

    L'objectif est de pouvoir lancer les PC sans avoir besoin de booter sur la clef (Ce qui peut entrainer des modification dans le BIOS) en utilisant seulement le réseau et un cable Ethernet comme matériel de plus nous pourront lancer plusieur installation en même temps.

    2. Déroulement

    1 - Installation du rôles WDS

    1.1 - Aller à la page d’ajout des rôles, cochez la case «Service de déploiement Windows».

    1.2 - A la demande d’ajout des fonctionnalités, cliquez le bouton « Ajouter des fonctionnalités ».

    1.3 - Laissez cocher les deux services de rôles associés à WDS.

    1.4 - Et pour terminer, vous pouvez cliquer sur Installer.

    2 - Configuration et démarage du serveur WDS

    2.1 - Lancer le service de déploiement Windows.

    2.2 - Et puis cliquer sur "Configurer le serveur"

    2.3 - Et laissez tous par défaut mis a part l'"Options d'installation" ou il faut mettre dans mon cas "Intégré à Active Directory"

    2.4 - Puis le démarrer

    3 - Image de démarrage et d'installation

    3.1 - Dans la console de gestion du service WDS, faites un clic droit sur le nœud « Images de démarrage » et cliquez sur « Ajouter une image de démarrage ».

    3.2 - Cliquez sur Parcourir et recherchez le fichier « boot.wim » qui est présent dans les fichier d'un ISO Windows puis choisir son nom

    3.3 - L’image de démarrage sera alors disponible dans la console de gestion de WDS.

    3.4 - Toujours dans la console de gestion de WDS, faites cette fois-ci un clic droit sur « Image d’installation » et cliquez sur « Ajouter une image d’installation ».

    3.5 - Il faut créer un groupe d’image pour stocker notre image d’installation puis recherchez le fichier d’installation .wim, après il faut choisir sa version puis son nom

    3.6 - Voila l'image dans le grouê et prêt a l'utilisation

    4 - Test

    4.1 - Dès la mise sous tension, appuyez sur la touche F12 (la plupart du temps) puis selectionner LAN, PXE ou Network

    4.2 - La machine va se faire affecter un IP par le serveur DHCP puis grace a ça vas communiquer avec le serveur PXE

    4.3 - Le client va télécharger localement l’image de démarrage (boot.wim) depuis le serveur pour initier le lancement de l’installeur.

    4.4 - Une fois téléchargé, le programme d’installation de système d’exploitation va démarrer. Il reste plus qu'a faire un installation windows normal.

    3. Conclusion

    En conclusion, l'implémentation d'un serveur PXE dans un environnement réseau est un choix stratégique qui offre une multitude d'avantages significatifs. Il simplifie grandement le déploiement et la gestion des systèmes d'exploitation sur un grand nombre de machines, permettant ainsi un gain de temps considérable et une réduction des coûts opérationnels.De plus si on utilise un ISO personaliser tout ce fait automatiquement
